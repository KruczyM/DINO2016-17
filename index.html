git config --global user.email ""

Komenda służy do konfiguracji emailu użytkownika. Jest to ważne ponieważ każda operacja zatwierdzenia w GIT korzysta z Nazwy użytkownika oraz Email. Dzięki opcji --global , wystarczy ze konfiguracja zostanie dokonana jednorazowo a GIT skorzysta z niej podczas każdej operacji wykonywanej przez Ciebie w danym systemmie.Jeśli zaistnieje potrzeba zmiany tych informacji dla konkretnego projektu, można skorzystać z git config bez opcji --global.

Przyklad: git config --global user.email jannowak@gmail.com



git commit -a -m ""

Komenda doda i zacommituje wszytkie zmodyfikowane pliki(te o których git nie wie nie sa zagrożone).
-m - dodaje dany komentarz do commita

Przyklad: git commit -a -m "System Chodzenia poprawiony"



git diff --staged

Komenda służy do zobaczenia zawartości poczekalni która trafi do repozytorium z najblizszym zatwierdzeniem. Samo polecenie git diff nie pokazuje wszystkich zmian dokonanych od ostatniego zatwierdzenia , tylko te które nie trafiły do poczekalni.



git checkout -b "branch"

Komenda służy do usuwania zmian w danym branchu.

Przyład: git checkout -b grazyna



git log --graph --all

Komenda pokaże nam logi commitów wszystkich gałęzi za pomocą wykresu.



git branch -d "branch"

Komenda usuwa branch. Zanim usuniemy brancha należy najpierw sprawdzić czy dany branch jest scalony do aktywnej gałęzi. Jeśli jest , to możemy usunąć dany branch , bez ryzyka ze stracimy jakieś dane.

Przyklad: git branch -d DanielBranch



git remote add "remote_name" "git url"

Komenda służy do dodawania zdalnych repozytoriów.

Przykład: git remote add Adrian git://github.com/Yetju000



git fetch "remote" "branch"

Komenda pobierze wszystkie dane z danego brancha zdalnego repozytorium.

Przyklad : git fetch Repo Daniel - pobierze wszystkie dane z brancha Daniel z reporytorium Repo.
